# VK Modifier

Инструмент пакетной обработки аудиофайлов. Изменяет цифровую структуру файлов для противодействия автоматическому распознаванию отпечатков — при этом звук на слух может оставаться практически неизменным в зависимости от выбранной агрессивности обработки. Работает через FFmpeg, интерфейс на PyQt6.

## Способы запуска

Программа доступна в двух вариантах:

- **Готовый EXE** — скачать `VK_Modifier.exe` и запустить. FFmpeg встроен, ничего дополнительно устанавливать не нужно. Только Windows 64-bit.
- **Из исходников** — требует Python, FFmpeg и установки библиотек. Работает на Windows, Linux, macOS.

---

## Вариант 1 — Готовый EXE (рекомендуется для Windows)

### Что скачать

Релиз версия скачать файл [VK_Modifier.exe](https://github.com/semnavmeleon/VK-Modifier/releases/tag/v1.1.0).

### Запуск

Дважды кликнуть по `VK_Modifier.exe`. При первом запуске Windows может показать предупреждение «Защитник Windows предотвратил запуск неизвестного приложения» — нажать **«Подробнее»** → **«Всё равно выполнить»**.

FFmpeg встроен в EXE — при распаковке он временно появляется в `%TEMP%\...` и удаляется при закрытии. Антивирус может сработать на это поведение — это ложное срабатывание, связанное с механизмом работы PyInstaller.

### Что делает программа при запуске

1. Распаковывает содержимое EXE во временную папку (`%TEMP%\_MEIxxxxxx`)
2. Добавляет эту папку в PATH, находит там `ffmpeg.exe` и `ffprobe.exe`
3. Запускает интерфейс
4. При закрытии временная папка удаляется автоматически

## Вариант 2 — Запуск из исходников

### Требования

| Компонент | Версия | Обязательно                                          |
| --------- | ------ | ---------------------------------------------------- |
| Python    | 3.10+  | да                                                   |
| PyQt6     | 6.4+   | да                                                   |
| mutagen   | любая  | да                                                   |
| FFmpeg    | 4.4+   | да (в PATH)                                          |
| pyqtgraph | любая  | нет (для визуализатора волны)                        |
| numpy     | любая  | нет (для визуализатора и спектрального маскирования) |

### Шаг 1 — Установить Python

Скачать с https://www.python.org/downloads/ и установить. **Обязательно** поставить галочку **«Add Python to PATH»** в самом начале установщика — без неё команды `python` и `pip` не будут доступны из терминала.

Проверка после установки — открыть новое окно командной строки и выполнить:

```
python --version
```

Должно вывести `Python 3.x.x`.

### Шаг 2 — Установить FFmpeg

FFmpeg — это отдельная программа обработки аудио/видео, которую использует VK Modifier. Нужно установить её так, чтобы она была доступна из командной строки по имени `ffmpeg`.

**Windows:**

1. Открыть https://ffmpeg.org/download.html, в разделе **Windows** перейти по ссылке на сборки от **gyan.dev** или **BtbN**
2. Скачать архив `ffmpeg-*-essentials_build.zip` (лёгкая версия) или full build
3. Распаковать архив. Внутри будет папка вида `ffmpeg-7.1-essentials_build` — переименовать её в `ffmpeg`
4. Переместить папку `ffmpeg` в корень диска C: — должно получиться `C:\ffmpeg`
5. Добавить `C:\ffmpeg\bin` в системную переменную PATH:
   - Нажать Win+R, ввести `sysdm.cpl`, нажать Enter
   - Вкладка «Дополнительно» → кнопка «Переменные среды»
   - В нижнем списке «Системные переменные» найти строку `Path`, дважды кликнуть
   - Нажать «Создать», ввести `C:\ffmpeg\bin`, нажать OK везде
6. Открыть **новое** окно командной строки (старое не увидит изменений) и проверить:
   
   ```
   ffmpeg -version
   ```
   
   Если вывелась информация о версии — установка прошла успешно.

**Linux:**

```
sudo apt install ffmpeg        # Debian / Ubuntu
sudo pacman -S ffmpeg          # Arch
sudo dnf install ffmpeg        # Fedora
```

**macOS:**

```
brew install ffmpeg
```

### Шаг 3 — Установить Python-библиотеки

Открыть командную строку и выполнить:

```
pip install PyQt6 mutagen pyqtgraph numpy
```

`pyqtgraph` и `numpy` необязательны, но без них не будет работать визуализатор формы волны и спектральное маскирование будет работать значительно медленнее.

### Шаг 4 — Запустить программу

Открыть командную строку в папке с файлами программы и выполнить:

```
python gui_qt.py
```

Или создать файл `run.bat` в папке программы с содержимым:

```
@echo off
python gui_qt.py
```

После этого можно запускать двойным кликом по `run.bat`.

## Первый запуск — пошаговый пример

Рассмотрим типичный сценарий: нужно обработать несколько MP3-файлов так, чтобы системы автоматического распознавания их не узнали, но на слух изменения были минимальны.

### 1. Добавить файлы

После запуска программы перетащить MP3-файлы из Проводника в **левую панель** — или нажать кнопку **«Добавить файлы»** (также `Ctrl+O`). Файлы появятся в списке, рядом с именем отобразится длительность.

При клике на файл в списке справа загружаются метаданные, обложка и форма волны (синяя кривая).

### 2. Выбрать метод обработки

В правой панели — семь вкладок с методами. Для минимально заметной обработки рекомендуется начать со **Спектрального джиттера**:

1. Перейти на вкладку **«Текстурные»**
2. Найти раздел **«Спектральный джиттер (notch-фильтры)»**
3. Поставить галочку **«Включить»**
4. Нажать кнопку пресета **«Мягкий»** — загрузятся 2 notch-фильтра с ослаблением 3–4 дБ
5. Режим автоматически переключится в **«Конструктор»**, где можно посмотреть и при необходимости изменить параметры каждого фильтра

Через 900 мс после последнего изменения параметра автоматически запустится **предпросмотр** на выбранном файле — в визуализаторе появится оранжевая кривая «после», и можно сравнить с исходной синей.

### 3. Настроить вывод

В разделе **«Настройки вывода»** (прямо под вкладками):

- Нажать **«Выбрать папку»** и указать, куда сохранять результаты. По умолчанию это `~/Desktop/Output` — папка создаётся автоматически.
- Выбрать **качество** — по умолчанию 320 kbps CBR. Можно оставить как есть.
- Шаблон имени файла находится на вкладке **«Имена»**. По умолчанию: `VK_{n:03d}_custom` → файлы будут называться `VK_001_custom.mp3`, `VK_002_custom.mp3` и т.д. Чтобы сохранить оригинальные имена — заменить шаблон на `{original}`.

### 4. Запустить обработку

Нажать **Запустить обработку** или `Ctrl+R`. В нижней части появится прогресс-бар и ETA. В логе будет отображаться статус каждого файла:

- зелёным — успешно обработан
- красным — ошибка (подробности в тексте лога)

После завершения можно нажать **«Открыть папку»** — откроется папка с результатами.

### 5. Проверить результат

Выбрать любой из исходных файлов в списке. В визуализаторе синяя кривая — исходник. Нажать кнопку **«Сбросить "после"»**, затем снова кликнуть на файл в списке и нажать `Ctrl+R` для повторной обработки (или открыть обработанный файл отдельно) — появится оранжевая кривая, и можно оценить разницу по RMS и Peak.

## Сохранение настроек (пресеты)

Если найдена удачная комбинация параметров, её можно сохранить:

1. Перейти на вкладку **«Имена»**
2. Нажать **«Сохранить текущие настройки»**, ввести имя (например, «Мягкий без изменений»)
3. Пресет появится в списке

Чтобы применить сохранённый пресет: выбрать его в списке и нажать **«Загрузить»**.

Все настройки также **автоматически сохраняются** при закрытии окна и восстанавливаются при следующем запуске — вручную ничего делать не нужно.

## Конвертер

Для конвертации между форматами без обработки: нажать **«Конвертер»** в верхней панели инструментов. Откроется отдельный режим, который использует тот же список файлов из левой панели.

1. Добавить файлы в левую панель (если не добавлены)
2. Выбрать выходной формат и качество
3. Указать выходную папку
4. Нажать **Запустить конвертацию**

Поддерживаемые выходные форматы: MP3, FLAC, WAV, OGG, AAC, M4A, WMA, Opus, AIFF, ALAC, WavPack, Monkey's Audio, TrueAudio, AC3, DTS, MP2, Musepack, Speex, AMR, AU, Matroska, Ogg FLAC, Core Audio, Shorten.

Для форматов с настраиваемым качеством (MP3, AAC, OGG, Opus, WMA) показывается список пресетов. Для FLAC, WAV, AIFF и других lossless-форматов параметры по умолчанию.

## Интерфейс — обзор панелей

### Левая панель — список файлов

- **Добавить файлы** — диалог выбора (`Ctrl+O`), или drag & drop из Проводника
- **Недавние** — список последних 50 добавленных файлов
- **Очистить** — удалить все файлы из списка (файлы на диске не трогает)
- Правая кнопка мыши на списке — контекстное меню с теми же действиями
- `Ctrl+A` — выделить все, `Delete` — удалить выделенные из списка

При клике на файл отображаются: метаданные (название, исполнитель, альбом, год, жанр, битрейт, частота), обложка, форма волны.

### Правая панель — Модификатор

**Вверху:** виджет обложки (можно загрузить свою или сгенерировать случайную), поля метаданных, информация об анализе трека.

**Вкладки методов:** Базовые, Спектральные, Текстурные, Продвинутые, Технические, Системные, Имена — каждый метод включается отдельной галочкой.

**Визуализатор формы волны:** синяя кривая — исходный файл, оранжевая — результат обработки. Показывает RMS и Peak в dBFS, дельту RMS между до и после.

**Прогресс-бар + ETA** — при обработке показывает прогресс и оставшееся время.

**Лог** — текстовый журнал с цветовой индикацией: серый — информация, зелёный — успех, оранжевый — предупреждение, красный — ошибка.

### Базовые

#### Спектральный джиттер — подробно

Два режима:

- **Случайный** — при каждом запуске случайно выбирает N частот из пула 120–12500 Гц и случайные ослабления до заданного максимума. Параметры: количество нотчей (1–16), максимальное ослабление (1–40 дБ).
- **Конструктор** — ручная настройка каждого фильтра: частота (20–20000 Гц), ослабление (1–40 дБ), ширина Q (0.5–10). Количество фильтров не ограничено, можно добавлять и удалять строки.

**Встроенные пресеты конструктора:**

| Пресет   | Фильтров | Диапазон частот    | Макс. ослабление | Характер                  |
| -------- | -------- | ------------------ | ---------------- | ------------------------- |
| Мягкий   | 2        | 630, 3150 Гц       | 4 дБ             | Практически незаметно     |
| Лёгкий   | 3        | 400, 1600, 6300 Гц | 6 дБ             | Едва слышимо              |
| Средний  | 5        | 250–12500 Гц       | 9 дБ             | Заметное изменение тембра |
| Сильный  | 6        | 120–10000 Гц       | 15 дБ            | Выраженное окрашивание    |
| Жёсткий  | 8        | 120–12500 Гц       | 22 дБ            | Сильное искажение спектра |
| Максимум | 11       | 120–12500 Гц       | 30 дБ            | Агрессивное подавление    |

#### VK Инфразвук — подробно

Генерирует сигнал в диапазоне 1–20 Гц и подмешивает его к основному аудио через `amix=inputs=2:duration=first:normalize=0`.

Стерео: левый канал — фаза 0°, правый — фаза π/6 (~30°, 0.5236 рад). Фазовое расхождение L/R сохраняется при mono-даунмиксе.

Адаптивная амплитуда: если включена, программа определяет пиковую амплитуду исходника и пропорционально уменьшает амплитуду инфразвука, не допуская перегрузки.

**Режимы:**

| Режим     | Описание                                                      |
| --------- | ------------------------------------------------------------- |
| simple    | Чистая форма волны без модуляции                              |
| modulated | Амплитудная модуляция огибающей                               |
| phase     | Фазовая модуляция                                             |
| harmonic  | Основной тон + обертоны h2/h3/h4 с настраиваемыми амплитудами |
| maximum   | Модулированный основной тон + модулированный второй обертон   |

**Формы волны:** `sine` — синусоида, `triangle` — треугольная, `square` — меандр.

**Параметры:** частота (1–20 Гц), амплитуда (0.01–1.0), частота модуляции (0.01–5 Гц), глубина модуляции (0–1), начальная фаза (0–2π рад), амплитуды гармоник h2/h3/h4.

### Продвинутые

| Метод                    | Параметры                            | Описание                                                                 |
| ------------------------ | ------------------------------------ | ------------------------------------------------------------------------ |
| Обрезка начальной тишины | макс. секунд, порог dBFS             | `silencedetect` определяет конец тишины в начале, затем `-ss` обрезает   |
| Вырезать фрагмент        | позиция % от длины, длина в секундах | Вырезает участок и склеивает оставшиеся части через `concat`             |
| Затухание в конце        | 0.5…60 с                             | `afade=t=out:st=X:d=N` — плавное затухание                               |
| Объединить с треком      | путь к файлу                         | Нормализует оба файла к 44100 Гц / PCM S16LE, склеивает через `concat`   |
| Сломанная длительность   | тип                                  | Записывает некорректный TLEN в ID3 и патчит frame count в Xing-заголовке |

Типы сломанной длительности: «Очень большая» (×50–200 от реальной), «Очень маленькая» (1–50 фреймов), «Случайная», «Максимум» (0xFFFFFF00 фреймов).

### Технические

| Метод                        | Описание                                                 |
| ---------------------------- | -------------------------------------------------------- |
| Фейковые метаданные          | Добавляет TXXX с `comment` из 100–500 случайных символов |
| Перемешать порядок ID3-тегов | `update_to_v23()` + `save()` через mutagen               |
| Джиттер битрейта             | Случайный битрейт из набора 192 / 224 / 256 / 320 kbps   |
| Удалить Xing/Info заголовок  | `-write_xing 0` — убирает VBR-заголовок                  |

### Системные

| Параметр                     | Описание                                                                                        |
| ---------------------------- | ----------------------------------------------------------------------------------------------- |
| Потоков                      | 1–16 параллельных потоков для пакетной обработки                                                |
| Задержка между потоками      | Пауза в секундах перед стартом каждого следующего потока                                        |
| Lossless промежуточные файлы | WAV вместо MP3 при нарезке/склейке — исключает повторное перекодирование. Включено по умолчанию |

### Имена файлов

Шаблон задаётся на вкладке **«Имена»**. Доступные переменные:

| Переменная   | Значение                            |
| ------------ | ----------------------------------- |
| `{n}`        | Порядковый номер (целое)            |
| `{n:03d}`    | Номер с ведущими нулями (001, 002…) |
| `{original}` | Исходное имя файла без расширения   |
| `{title}`    | Тег Title из метаданных             |
| `{artist}`   | Тег Artist                          |
| `{album}`    | Тег Album                           |
| `{year}`     | Тег Year                            |

По умолчанию: `VK_{n:03d}_custom` → `VK_001_custom.mp3`, `VK_002_custom.mp3`.
Чтобы сохранить оригинальные имена: заменить на `{original}`.

## Настройки вывода

Находятся под вкладками в панели модификатора:

- **Выходная папка** — по умолчанию `~/Desktop/Output`, создаётся автоматически
- **Сохранять оригинальные теги** — переносит ID3-теги из исходника в результат
- **Сохранять оригинальную обложку** — переносит APIC-обложку из исходника
- **Рандомные названия в тегах** — заменяет тег Title на случайное сочетание типа «Wave 247»
- **Удалять оригиналы после обработки** — необратимо, требует отдельного подтверждения
- **Качество** — CBR 128/192/256/320 kbps или VBR Q0–Q6 (LAME)

## Частые проблемы

### При запуске из исходников

| Симптом                                        | Причина                   | Решение                                                                |
| ---------------------------------------------- | ------------------------- | ---------------------------------------------------------------------- |
| `'python' is not recognized`                   | Python не в PATH          | Переустановить Python с галочкой «Add Python to PATH»                  |
| `ModuleNotFoundError: No module named 'PyQt6'` | Библиотеки не установлены | `pip install PyQt6 mutagen pyqtgraph numpy`                            |
| В тулбаре «FFmpeg: ✗ не найден»                | ffmpeg не в PATH          | Установить FFmpeg, добавить `bin`-папку в PATH, открыть новый терминал |
| Форма волны не отображается                    | нет pyqtgraph или numpy   | `pip install pyqtgraph numpy`                                          |
| Спектральное маскирование медленно работает    | нет numpy                 | `pip install numpy` — без него 27 запусков FFmpeg вместо одного FFT    |

### При использовании EXE

| Симптом                                              | Причина                                                                                                                | Решение                                                                                    |
| ---------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| Антивирус блокирует файл                             | PyInstaller распаковывает файлы во временную папку — некоторые антивирусы расценивают это как подозрительное поведение | Добавить исключение в антивирусе или запустить из папки, не проверяемой в реальном времени |
| Windows заблокировал запуск («Неизвестный издатель») | Файл не имеет цифровой подписи                                                                                         | Нажать «Подробнее» → «Всё равно выполнить»                                                 |
| Долгий первый запуск (10–30 сек)                     | PyInstaller распаковывает ~200 MB во временную папку                                                                   | Это нормально, последующие запуски быстрее при уже существующей временной папке            |
| В тулбаре «FFmpeg: ✗ не найден»                      | Временная папка ещё не создана или антивирус удалил ffmpeg.exe из неё                                                  | Закрыть программу, временно отключить антивирус, снова запустить                           |

### Общие

| Симптом                                       | Причина                                      | Решение                                                                             |
| --------------------------------------------- | -------------------------------------------- | ----------------------------------------------------------------------------------- |
| Файл после обработки слишком сильно изменился | Агрессивные настройки                        | Использовать пресет «Мягкий» или уменьшить параметры вручную                        |
| Ошибка при обработке конкретного файла        | Повреждённый файл или редкий формат          | Проверить лог, попробовать предварительно конвертировать файл в WAV через конвертер |
| Обработка одного файла занимает очень долго   | Включено спектральное маскирование без numpy | `pip install numpy` резко ускорит анализ                                            |
